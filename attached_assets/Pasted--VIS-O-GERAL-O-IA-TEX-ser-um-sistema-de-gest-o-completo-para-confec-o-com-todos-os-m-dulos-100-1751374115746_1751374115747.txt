💻 MELHOR STACK DE TECNOLOGIA (FINAL E ESCALÁVEL)
🔧 Stack Recomendada
Camada	Tecnologia	Por que usar?
Frontend	Next.js (React) + Tailwind CSS	Layout moderno, rápido, SEO, componentes reutilizáveis
Componentes UI	shadcn/ui ou Radix + Tailwind	Componentes elegantes e com modais/dialogs prontos
Animações	Framer Motion	Transições suaves (cards, modais, drag-and-drop)
Backend	FastAPI (Python)	Rápido, leve, perfeito para APIs e escalável
Banco de Dados	PostgreSQL	Robusto, relacional e preparado para crescimento
ORM	SQLAlchemy ou Tortoise ORM	Fácil de usar com FastAPI
Autenticação	JWT + OAuth2 (FastAPI Users)	Login seguro, roles, múltiplos perfis
Exportações	fpdf2 (PDF) + pandas/xlsxwriter (Excel)	Suporte completo a exportações profissionais
Armazenamento	Amazon S3 ou local	Para imagens, moldes, vídeos etc.
Backend Real-Time (opcional)	WebSockets / Socket.IO	Para rastreamento ao vivo na produção

🧵 2. TECIDOS
🎯 Objetivo
Gerenciar os tecidos da confecção com total controle de estoque, composição, rendimento e precificação, além de integrar com os módulos de Precificação, Produção e Compras.
________________________________________
🧱 Estrutura Visual
🔸 A. Topo do Módulo (estilo IA.TEX)
•	📦 Total de Tecidos
•	🟠 Baixo Estoque
•	💰 Valor Total em Estoque (R$)
•	Botão 🔴 “+ Novo Tecido” (abre modal)
🔸 B. Cards Estilo Trello (por tecido)
Cada card mostra:
•	Miniatura da imagem do tecido (ou textura)
•	Nome (abaixo da imagem)
•	Preço por kg (calculado automaticamente)
•	Rendimento médio
•	Tag de status:
o	🟢 Disponível
o	🔴 Baixo Estoque
•	Clique abre modal completo
🔸 C. Modal: Cadastro/Edição de Tecido
Campos obrigatórios:
•	Nome
•	Tipo (Dry, Suplex, etc.)
•	Cor
•	Composição
•	Gramatura (g/m²)
•	Largura útil (cm)
•	Preço por metro
•	Fornecedor
•	Estoque atual (em metros)
•	Imagem (upload)
Campos calculados:
•	Preço por kg = preço / (gramatura * largura / 10.000)
•	Rendimento estimado = função do peso da peça / consumo
•	Classificação automática: Baixo Estoque (< 20m)
Ações:
•	✅ Salvar
•	🔄 Atualizar estoque
•	🗑️ Excluir
________________________________________
🔁 Integrações
•	Precificação: ao selecionar tecido, importa todos os dados técnicos
•	Produção: calcula necessidade de rolos por modelo/pedido
•	Financeiro: usa preço por metro para calcular gasto por pedido
•	Estoque: alerta de baixo estoque com notificação visual

•	Dropdown com tipos existentes (ex: Top, Calça)

🟪 Etapa 3: Escolha do Tecido
•	Lista de tecidos cadastrados (em card)

o	Puxa automaticamente gramatura, largura, rendimento
o	Inserir % de desperdício
________________________________________
🟩 Etapa 4: Pesos por Tamanho
•	Tamanhos padrão (P, M, G, GG, XG)
•	Campo para peso da peça por tamanho (em gramas)
•	Calcula automaticamente o consumo de tecido com base na gramatura e rendimento

🟥 Etapa 5: Custos por Etapas (cards agrupados)
5A. Criação
•	Ficha técnica, modelagem, piloto
5B. Itens (Aviamentos e Materiais)
•	Zíper, Elástico, Etiqueta, Embalagem
•	Cada item com:
o	Descrição
o	Valor unitário
o	Quantidade
o	% de desperdício
o	Unidade (m, un, kg...)
5C. Mão de Obra
•	Costura, Prensa, Embossar, Dobrar
•	Mesmo formato do grupo anterior
5D. Gerenciais
•	Energia, aluguel, transporte, marketing, comissão
•	Podem ser % ou valores fixos por peça
________________________________________
🟫 Etapa 6: Simulação de Preço
•	Calcula:
o	Custo total por tamanho
o	Margem configurável
o	Preço de venda (atacado, varejo)
o	Markup e lucro bruto
•	Exibe tabela com:
o	Preço sugerido por tamanho
o	Lucro estimado por peça
o	Média geral e total do lote
•	Opção: “Simular com outro tecido” → sistema recalcula peso e custo com o novo
________________________________________
🟧 Etapa 7: Salvar & Exportar
•	✅ Salvar como Modelo
•	🧩 Salvar como Template (com nome próprio)
•	Exportar como:
o	PDF técnico (visual refinado com imagem ao lado)
o	Futura versão em imagem
•	Gera e salva os seguintes arquivos:
o	modelos.csv
o	pesos_<modelo>.csv
o	etapas_<modelo>.csv
o	simulacao_<modelo>.csv

🔁 Integrações
•	Modelos: modelo salvo vai para a aba de visualização e produção
•	Pedidos: modelo pode ser puxado direto para orçamentos

•	Estoque: define consumo previsto de tecido por pedido

from fastapi import Depends, HTTPException
from fastapi.security import OAuth2PasswordBearer
from jose import JWTError, jwt

SECRET_KEY = "your-secret-key"
ALGORITHM = "HS256"

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

def get_current_user(token: str = Depends(oauth2_scheme)):
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        return payload
    except JWTError:
        raise HTTPException(status_code=401, detail="Invalid token")

from fastapi import APIRouter, Depends
from sqlalchemy.orm import Session
from backend.database import get_db
from backend.models import Tecido

router = APIRouter()

@router.post("/tecidos")
def criar_tecido(tecido: TecidoCreate, db: Session = Depends(get_db)):
    db_tecido = Tecido(**tecido.dict())
    db.add(db_tecido)
    db.commit()
    return {"message": "Tecido criado com sucesso!"}
